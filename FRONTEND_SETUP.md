## 🚀 如何使用

### 1. 启动前端（本地开发）

```bash
cd frontend
npm start
```

前端会在 `http://localhost:3001` 启动

### 2. 访问采购清单

登录后，点击左侧菜单的 **📦 采购清单**

### 3. 操作流程

#### 添加物品：
1. 点击右上角 **+ 添加物品** 按钮
2. 填写表单：
   - 物品名称 *（必填）
   - 分类 *（必填）：会亲家/订婚/结婚/其他
   - 金额
   - 数量
   - 单位
   - 状态：待购买/已购买/已取消
   - 优先级：高/中/低
   - 购买日期
   - 购买商店
   - 备注
3. 点击 **添加** 按钮

#### 查看物品：
- 物品按分类分组显示
- 每个分类显示统计信息（项目数、已购买、待购买、总金额）
- 可以使用顶部筛选器过滤

#### 更新状态：
- 直接在物品卡片上选择新状态
- 自动保存

#### 编辑物品：
- 点击物品卡片上的 ✏️ 按钮
- 修改信息
- 点击 **更新** 按钮

#### 删除物品：
- 点击物品卡片上的 🗑️ 按钮
- 确认删除

## 🎨 UI 特点

### 1. 统计卡片
- 渐变色背景，美观醒目
- 实时显示统计数据
- hover 有动画效果

### 2. 分类分组
- 按分类自动分组
- 每组显示汇总信息
- 图标区分不同分类

### 3. 优先级标识
- 红色（高）、蓝色（中）、灰色（低）
- 一目了然的优先级

### 4. 状态管理
- 下拉快速切换状态
- 已购买项目半透明显示
- 状态颜色编码

### 5. 响应式设计
- 适配桌面、平板、手机
- 移动端优化布局

- 统计卡片两列显示（小屏单列）

## 🎯 使用场景示例

### 会亲家准备
```
1. 点击"+ 添加物品"
2. 选择分类"会亲家"

4. 设置高优先级
5. 购买后标记为"已购买"
```

### 订婚筹备
```
1. 添加订婚相关物品
2. 设置优先级
3. 逐步购买并标记
```

```
2. 按优先级排序

4. 控制预算
```

### 1. 构建前端
```bash

npm run build
```

```bash

scp -r build/* root@服务器IP:/www/wwwroot/accounting-system/frontend/build/

./deploy.sh
```

### 3. 配置 Nginx
前端文件会被 Nginx 提供服务（已配置）
